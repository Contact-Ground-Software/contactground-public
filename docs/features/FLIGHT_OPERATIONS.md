# Flight Operations

This manual explains post-flight update workflows and operations handling.

## 1. What Flight Operations Covers

Flight operations captures post-flight usage data and operational notes, including:
1. Hours flown
2. Current hours (and Hobbs, if enabled)
3. Oil added
4. Squawks
5. Pilot reports/notes

## 2. Member Post-Flight Workflow

1. Open your post-flight update link or flight ops update page.
2. Enter hours and related details.
3. Report any squawks.
4. Submit.

This keeps resource time, maintenance visibility, and billing inputs accurate.

## 3. Operations Team Workflow

In `Flight Ops` management pages, operations staff can:
1. Review submitted updates.
2. Correct data quality issues. Use `Messaging` or `News` to let members know your data quality expectations.
3. Add manual entries when needed.
4. Remove incorrect entries.

## 4. Why Data Quality Matters

Accurate flight ops updates directly affect:
1. Maintenance timing and alerts
2. Resource hours accuracy
3. Invoice accuracy
4. Operations reporting

## 5. Best Practices

1. Submit updates as soon as practical after flight.
2. Record squawks clearly and specifically.
3. Use `Flight Ops` page to fix anomalies before billing cycles.
