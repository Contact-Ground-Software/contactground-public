# Organizational Messaging

This manual explains how to send organization-wide messages to members.

## 1. What Organizational Messaging Does

`Messaging` lets you send a single announcement to a targeted member group without building separate email lists manually.

## 2. Before You Start

Your organization must enable messaging in organization settings. Enabled targeting options can include:
1. Individuals
2. Roles
3. Endorsements
4. Entire organization

The available options you see on the `Messaging` page depend on what your organization enabled.

## 3. Sending A Message

1. Open `Messaging` from the sidebar.
2. In `Destination`, choose the messaging mode (if multiple modes are enabled).
3. Select recipients based on the chosen mode.
4. Enter a subject.
5. Enter message body.
6. Send.

## 4. Targeting Modes

1. `Individuals`:
Pick one or more specific members.
2. `Roles`:
Send to one or more roles (for example all instructors or all operations members).
3. `Endorsements`:
Send to members who hold selected endorsements.
4. `Entire Organization`:
Send to all current members.

## 5. Delivery And Results

After sending, the page shows how many recipients were reached.

Delivery uses in-app notifications combined with each recipient's personal notification settings (email/SMS/push preferences).

## 6. Reviewing Replies

Owners and admins can open `Review replies` from the messaging page to view:
1. Sent message threads
2. Replies received from members

## 7. Best Practices

1. Use clear subjects so members can triage quickly.
2. Keep one topic per message.
3. Use role- or endorsement-based targeting for operational updates.
4. For policy changes, send to the entire organization and include an effective date.
5. Avoid profanity, irrelevant content, and keep all message text within acceptable organizational guidelines.
